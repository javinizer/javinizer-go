# Quick Task: Actress DB Enrichment Gaps

## Context

From debug session `actress-db-data-not-in-output`, three gaps were identified where actress database data (especially user-updated ThumbURL) does not flow into organized output:

1. **Aggregator has no actress DB enrichment** — scrapers that return actresses without ThumbURL don't get it filled from the DB
2. **NFO merge can override DB data in update mode** — NFO actresses (stale ThumbURL) replace DB actresses before Upsert
3. **No explicit "enrich from DB" step** — enrichment is incidental via Upsert, could be lost if pipeline changes

## Tasks

### Task 1: Add `EnrichActressesFromDB` function to worker package

**files:**
- internal/worker/actress_enrich.go (new)
- internal/worker/actress_enrich_test.go (new)
- internal/database/actress_repo.go

**action:**
Create a new file `internal/worker/actress_enrich.go` with a function `EnrichActressesFromDB` that:
1. Takes a `*models.Movie`, an `*database.ActressRepository`, and a `*config.Config`
2. For each actress in `movie.Actresses`, looks up the actress database by DMMID first, then JapaneseName, then FirstName+LastName
3. If a DB match is found, fills in missing fields from the DB record (ThumbURL, FirstName, LastName, JapaneseName) — does NOT overwrite fields that already have values
4. If `actress_database.enabled` is false in config, returns immediately (no-op)
5. Returns the count of actresses enriched (for logging)

Add a `FindByDMMID` method to `ActressRepository` if it doesn't exist yet (needed for DMMID-based lookup).

**verify:**
- `EnrichActressesFromDB` correctly enriches actresses with missing ThumbURL from DB
- Does not overwrite existing non-empty fields
- Returns early when `actress_database.enabled` is false
- Handles case where no DB match is found (actress unchanged)
- Tests cover: enrichment by DMMID, by JapaneseName, by FirstName+LastName, no-overwrite, disabled config, no match

**done:**
`go test -v -run TestEnrichActressesFromDB ./internal/worker/`

### Task 2: Wire enrichment into pipeline — after aggregation, after NFO merge

**files:**
- internal/worker/single_scrape.go
- internal/worker/scrape_cache.go
- internal/worker/tasks.go

**action:**
1. In `single_scrape.go` `RunBatchScrapeOnce`: Call `EnrichActressesFromDB` after aggregation (line ~190, after `movie.OriginalFileName` assignment) AND after NFO merge (line ~193, after `mergeScrapedNFO` returns). This ensures:
   - Even if scrapers don't provide ThumbURL, the DB fills it in
   - After NFO merge potentially overrides actress data, the DB re-enriches missing fields

2. In `scrape_cache.go` `handleCacheHit`: Call `EnrichActressesFromDB` after `mergeCachedNFO` (line ~60) on the `movieToReturn`. This ensures update mode with cached movies also gets DB enrichment.

3. In `tasks.go`: Ensure `ActressRepository` is accessible where `EnrichActressesFromDB` is called. The `BatchJob` or the calling context likely already has DB access — check how `movieRepo` is passed and obtain `actressRepo` from the same `database.DB` instance.

Add logging: `logging.Debugf("[Batch %s] File %d: Enriched %d actresses from database", job.ID, fileIndex, enriched)`

**verify:**
- Normal scrape path: actresses get enriched from DB after aggregation
- Update mode with NFO merge: actresses get re-enriched from DB after NFO merge
- Cache hit + update mode: actresses get enriched from DB after cached NFO merge
- No duplicate enrichment (two calls are fine — second is a no-op if first filled everything)
- `go build ./...` passes
- Existing tests still pass

**done:**
`go test -short ./internal/worker/ && go build ./...`
